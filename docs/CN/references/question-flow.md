# HarnessStack 问题流

## 用途

在生成契约文档前，使用本参考执行精简问题流。

## 长效问题

仅在以下情况询问：

- 仓库没有 `docs/HarnessStack/longterm.md`，或
- 有证据表明长效基线需要完整重写

推荐问题：

1. 本仓库有多少活跃贡献者？
2. 本仓库主要是产品、研究还是平台类？
3. 本仓库预期短命还是长期维护？
4. 仓库中是否已有稳定的长效工作流基线？

## 临时任务问题

针对当前任务：

1. 需求是 `clear`、`partially-clear` 还是 `vague`？
2. 交付目标是什么？
3. 风险等级是什么？
4. 本任务是否会产生应在任务结束后仍保留的知识？

## 输出决策规则

### 仅输出临时契约

当：

- 已存在 `docs/HarnessStack/longterm.md`，且
- 未触发长效重写条件

### 同时输出长效与临时契约

当：

- 不存在 `docs/HarnessStack/longterm.md`，或
- 触发了长效重写条件

### 输出长效重写建议而非直接重写

当：

- 仓库已有稳定的长效契约，且
- 重写证据薄弱或不完整

## 输出形态

长效输出：

- 产出完整的 `docs/HarnessStack/longterm.md`
- 包含来源模板与当前生效基线

临时输出：

- 产出完整的 `docs/HarnessStack/temporary-<task>.md`
- 包含任务评估、调整、最终生效栈与退出规则

二者共通：

- 避免仅指向模板
- 避免残缺笔记
- 写出最终生效契约
