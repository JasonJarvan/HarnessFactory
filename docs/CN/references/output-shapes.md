# HarnessStack 输出形态

## 用途

在问题流结束后，使用本参考保持输出一致。

## 输出模式

### 模式 A：仅临时契约

适用于：

- 已存在 `docs/HarnessStack/longterm.md`，且
- 未触发长效重写条件

必须输出：

- 一份完整的 `docs/HarnessStack/temporary-<task>.md`

### 模式 B：长效契约与临时契约

适用于：

- 不存在 `docs/HarnessStack/longterm.md`，或
- 明确触发了长效重写条件

必须输出：

- 一份完整的 `docs/HarnessStack/longterm.md`
- 当有当前任务时，另有一份完整的 `docs/HarnessStack/temporary-<task>.md`

### 模式 C：长效重写建议（非直接重写）

适用于：

- 当前证据表明长效基线可能有问题，但
- 信息尚不足以安全重写

必须输出：

- 在对话中给出建议
- 可选：为当前任务提供临时契约

在此模式下**不得**静默改写 `longterm.md`。

## 长效输出形态

生成的 `longterm.md` 须包含：

1. 来源模板与更新元数据
2. 长效仓库评估
3. 完整的五层生效基线
4. 临时契约边界规则
5. 完整重写条件

## 临时输出形态

生成的 `temporary-<task>.md` 须包含：

1. 任务元数据
2. 任务评估
3. 相对长效基线的调整
4. 最终生效的五层栈
5. 执行规则
6. 与长效基线的冲突规则
7. 退出与合并规则

## 完整性规则

- 写出完整生效文档，而非指针或引用。
- 不要只输出「使用模板 X」。
- 不要让关键工作流层级隐含不明。
- 若某层有意不启用，须明确写出。
